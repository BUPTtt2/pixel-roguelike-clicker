## 核心玩法亮点

### 武器晋升系统
5种武器 × 3阶段 × 10级，Lv4 和 Lv7 为质变节点：

| 武器 | 基础 | 进阶(Lv4) | 终极(Lv7) |
|------|------|-----------|-----------|
| 暗影剑 | 流血吸血 | 流血叠加3层 | 连击血怒+血爆 |
| 烈焰斧 | 燃烧AOE | 燃烧叠加5层 | 火域引爆 |
| 寒冰法杖 | 冰锥减速 | 冰锥分裂 | 冰封万物秒杀 |
| 风暴弓 | 暴击穿透 | 暴击双发 | 标记+全穿透 |
| 奥术魔杖 | 元素共鸣 | 奥术印记 | 元素风暴 |

### Boss战利品经济
- 击败Boss后开放战利品商店（9卡片可重复购买）
- 金币仅Boss战后可消费 — 小怪不掉金币，Boss金币+50%
- 元素材料系统：碎片 / 尘埃 / 结晶，用于武器晋升
- 每次Boss战解锁新武器（斧→法杖→魔杖）

### 深度构筑
- 24种专精分6系（攻击/防御/经济/元素/召唤/特殊）
- 经验曲线 1.35^level 指数增长 — 越后期越难升
- 随从系统：5种元素随从协同作战

## AI 协作开发过程

本项目从设计到实现全程使用 AI（Claude + Trae IDE）协作完成。

### AI 承担的工作

| 模块 | AI 的角色 |
|------|-----------|
| 系统设计 | AI 主导设计武器晋升机制、经济体系、经验曲线，人工提供方向反馈 |
| 编码实现 | AI 生成全部 4900+ 行游戏代码，人工零手写 |
| 数值平衡 | AI 计算经验曲线/金币掉落/武器伤害成长表，人工试玩验证 |
| Bug修复 | AI 自主定位双武器等级系统冲突、经验共享等隐蔽bug |
| 迭代优化 | 多轮对话迭代：用户反馈"升级太快"→AI重算曲线；"武器太弱"→AI重构晋升 |

### AI 的核心价值

1. **方案取舍**：原方案金币可买经验，用户指出不合理，AI 重新设计为"金币仅Boss战后消费"的闭环经济
2. **快速迭代**：从 v1 到 v5，每轮重构AI在分钟级完成代码改写，人工只需试玩验收
3. **系统性思考**：AI 主动发现 weaponLevel 标量与 weaponLevels 对象混用的架构缺陷并统一修复

## 技术架构

- **引擎**：Phaser 3.60（CDN加载，无构建步骤）
- **架构**：单文件 game_v5.js（4900+行），纯前端零后端
- **存档**：localStorage 自动保存进度/解锁/成就
- **美术**：纯代码绘制像素风（无外部图片素材）
- **音效**：Web Audio API 程序化生成
