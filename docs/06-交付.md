## 二、设计说明

### 核心架构

```
index.html
  └── game_v5.js（单文件，约 4900 行）
        ├── 常量配置层：COLORS / BossConfigs / EnemyConfig
        ├── 实体类层：Weapon / Enemy / Boss / Projectile / Minion / PickupItem / Merchant
        ├── 系统类层：EnemyManager / BossManager / ParticleSystem / AudioSystem
        ├── 数据层：RuntimeData（玩家状态）/ SaveData（持久化）
        ├── 专精系统：24种专精 × 6系
        └── 场景层：GameScene（主场景，含所有UI与游戏逻辑）
```

### 核心设计决策

**1. 武器晋升机制（3阶段×10级）**

5种武器各有独立成长线，Lv4/Lv7为质变节点：
- 剑：流血叠加 → 连击血怒 → 血爆
- 斧：燃烧叠加 → 火域 → 火域引爆
- 法杖：冰锥分裂 → 冰冻领域 → 冰封万物秒杀
- 弓：暴击双发 → 标记系统 → 全穿透
- 魔杖：元素共鸣 → 奥术印记 → 元素风暴

晋升需消耗元素材料（碎片/尘埃/结晶），材料来自Boss掉落。

**2. 金币经济体系**

- 小怪金币产出归零（仅精英/宝箱怪给少量金币）
- Boss金币提升50%
- 金币唯一消费渠道：Boss战利品商店（9卡片网格）
- 暂停商店已移除，避免随时消费破坏稀缺感

**3. 经验曲线**

- 玩家经验：`60 × 1.35^(level-1)`，后期明显越来越难
- 武器经验：分段表 `[30,50,80,120,180,260,380,550,800]`
- 升级到Lv3/Lv6时需材料才能突破

**4. 元素效果系统**

- 流血/燃烧：可叠加层数，每tick造成伤害+视觉粒子
- 减速/冰冻/击晕：控制效果+对应颜色粒子
- 护甲穿透：弓专属，直接实现到 `takeDamage` 减伤计算
- 标记/火域/冰冻领域/奥术印记：高级武器机制

**5. 专精系统**

24种专精分6系（战斗/防御/元素/暗影/增益/特殊），每系4种，提供被动加成。Boss商店可购买随机专精。

## 六、作品说明（按要求.md格式）

### 作品名称
**像素地牢猎手（Pixel Dungeon Hunter）**

### 解决的核心业务痛点
泛用户在休闲场景下缺乏低门槛、高重玩性的轻量游戏。本作以"点击即玩"的极简操作降低准入门槛，以肉鸽随机构筑+武器晋升深度保证数小时重玩价值，填补碎片化娱乐场景。

### 核心玩法/功能亮点
1. **点击挂机战斗**：全屏点击造成范围伤害，敌人自动AI进攻
2. **5武器×3阶段晋升**：每把武器10级成长+2次质变（Lv4/Lv7），元素机制各异
3. **Boss战利品商店**：金币唯一消费渠道，9卡片可重复购买，武器晋升需元素材料
4. **元素材料经济**：碎片/尘埃/结晶三级材料，小怪稀有掉落+Boss固定掉落
5. **24专精构筑**：6系专精自由搭配，每局Build不同
6. **元素视觉反馈**：流血/燃烧/冰冻/减速各有专属粒子效果

### 项目创意设计
- **晋升机制**：将传统"升级加数值"升级为"阶段质变"，让每次晋升有玩法变化而非数字增长
- **金币稀缺化**：移除小怪金币产出和暂停商店，让Boss战利品商店成为唯一消费点，强化Boss战意义
- **经验曲线递增**：1.35倍指数曲线+分段武器经验表，实现"越后期越难升"的肉鸽节奏

### AI协作开发过程
- **代码生成**：AI完成全部4900行游戏代码，含战斗系统/Boss AI/UI/存档/音效
- **功能迭代**：AI根据玩家反馈多轮迭代，修复双武器等级系统Bug、重构经济体系、新增武器晋升机制
- **方案取舍**：AI在"金币商店时机"上提出Boss后开放方案（替代暂停商店），在"晋升材料来源"上设计三级掉落体系，平衡了稀缺性与可玩性
- **质量保障**：AI自主执行语法检查、Grep验证改动完整性、对照需求文档逐项核对功能点
