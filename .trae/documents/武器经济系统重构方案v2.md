# 武器经济系统重构方案 v2

## 摘要

针对用户反馈：武器太弱无晋升感、金币买经验不合理、元素特性看不出、小怪掉落不合理、升级不够难。本方案彻底重构武器晋升机制（5武器×3阶段×10级）、金币经验体系（金币稀缺+Boss商店）、掉落系统（新增元素材料）、元素效果（数值+视觉+护甲穿透实现），并修复4个关键Bug。

所有改动集中在 `d:\Appt\大三下\学习\vue,fastAPI,langchain\game_v5.js`。

## 一、当前状态分析

### 1.1 武器系统问题

- **Weapon类（L108-203）**：5武器各有10级，但成长线性（剑+8/级、斧+18/级），**无质变节点**
- **getElementEffect（L3597-3622）**：元素数值微弱（剑流血dps 0.25+0.05*lvl，5级才0.5）
- **弓护甲穿透Bug**：L3612设置`effect.armorPierce`，但L1000-1026的`Enemy.takeDamage`未读取，L3690-3710的`applyElementEffect`也未传入
- **武器描述（L207-211）**：5武器无阶段标识，玩家无晋升感

### 1.2 双武器等级系统Bug（严重）

存在两套不互通的系统：

| 系统 | 类型 | 位置 | 用途 |
|------|------|------|------|
| `runtime.weaponLevel` | 标量 | L2006, L4067同步, L4279/4318 Boss奖励, L3568/3813/1426/4454/4706 伤害/HUD/统计 | 部分使用 |
| `scene.weaponLevels[type]` | 对象 | L2339/2919/2981, L3787 addWeaponExp, L680/691/3504/3983 攻击计算 | 部分使用 |

结果：武器经验升级后，HUD/伤害计算可能仍用旧等级。

### 1.3 weaponExp全局共享Bug

L3790：`this.weaponExp = (this.weaponExp || 0) + amount` —— 所有武器共用一个标量，切换武器后经验错乱。

## 二、武器晋升机制设计

### 2.1 通用规则

- **3阶段**：基础（Lv1-3）/ 进阶（Lv4-6）/ 终极（Lv7-10）
- **2个晋升点**：Lv4（基础→进阶）、Lv7（进阶→终极）
- **晋升材料**：基础→进阶需 3元素碎片 + 1奥术尘埃；进阶→终极需 2元素结晶 + 5元素碎片
- **晋升触发**：Boss战利品商店中点击"武器晋升"卡片
- **字段**：`runtime.weaponPromotions[type]`（0/1/2）；`runtime.elements.{shard, dust, crystal}`

```javascript
getPromotionStage(type) { return this.runtime.weaponPromotions[type] || 0; }
getDamageGrowth(level) {
    const stage = this.getPromotionStage(this.id);
    const base = { sword:[8,12,15], axe:[18,22,25], staff:[5,8,10], bow:[10,12,15], wand:[4,6,8] };
    const arr = base[this.id] || base.sword;
```

```javascript
getElementEffect(wlvl) {
    const w = this.runtime.weapon;
```

```javascript
takeDamage(amount, isCrit = false, armorPierce = 0) {
    if (!this.alive) return false;
    let dmg = amount;
    if (this.config.armor) {
        const reduction = 0.4 * (1 - armorPierce);  // armorPierce=1时无视护甲

    }

}
```

```javascript
    if (this.freeze) { enemy.applyFreeze(1500); this.scene.particles.freezeEffect(this.screenX(), this.screenY()); }
    if (this.burn) enemy.applyBurn(Math.floor(dmg * 0.2), 3000);
    this.applyElementEffect(enemy, dmg);
    if (died) this.scene.addWeaponExp(this.scene.runtime.weaponType, 2);  // 新增
}
```

```javascript
addWeaponExp(weaponType, amount) {
    if (!this.weaponLevels[weaponType]) this.weaponLevels[weaponType] = 1;
    const cur = this.weaponLevels[weaponType];
    if (cur >= 10) return;
    if (!this.weaponExp[weaponType]) this.weaponExp[weaponType] = 0;
    this.weaponExp[weaponType] += amount;
    const needTable = [30, 50, 80, 120, 180, 260, 380, 550, 800];
    const need = needTable[cur - 1] || 999;
    if (this.weaponExp[weaponType] >= need) {
        // 检查晋升材料（Lv3→4和Lv6→7）
        if (cur === 3) {
            const e = this.runtime.elements;
            if (e.shard < 3 || e.dust < 1) {
                this.showCenterText(`需要 3元素碎片+1奥术尘埃 晋升！`, '#ff8844', 1500);
                return;
            }
        }
        if (cur === 6) {
            const e = this.runtime.elements;
            if (e.crystal < 2 || e.shard < 5) {
                this.showCenterText(`需要 2元素结晶+5元素碎片 晋升！`, '#ff8844', 1500);
                return;
            }
        }
        this.weaponExp[weaponType] -= need;
        this.weaponLevels[weaponType] = Math.min(10, cur + 1);
        if (cur === 3) this.runtime.weaponPromotions[weaponType] = 1;
        if (cur === 6) this.runtime.weaponPromotions[weaponType] = 2;
        this.showCenterText(`⚔ ${WeaponFactory.info(weaponType).name} Lv.${this.weaponLevels[weaponType]}!`,
            WeaponFactory.info(weaponType).color, 1000);
        this.particles.explosion(GW / 2, GH / 2, 40, WeaponFactory.info(weaponType).color);
        this.audio.play('levelup');
    }
}
```

## 八、假设与决策

### 假设

1. 用户接受"武器晋升需要材料"的设计（用户原话"要有晋升机制"未明确是否要材料，但材料系统让晋升有重量感）
2. 用户接受"Boss战利品商店"替代"3选1免费奖励"（用户原话"只能在boss战之后开放购买"明确指向Boss后商店）
3. 用户接受"暂停商店完全移除"（用户原话"金币买经验卷轴这些都太不合理了，不要"指向暂停商店卖消耗品不合理）
4. 5种武器的晋升设计基于其元素特性（剑流血/斧燃烧/杖冰冻/弓穿透/魔杖弹射），与现有元素系统一致

### 决策

1. **保留免费奖励**：Boss商店中保留"三选一免费奖励"（武器经验+5/生命恢复/新随从），让玩家不至于完全无免费收益
2. **不删除Merchant**：Merchant仍有用（卖非经验/非金币类消耗品+元素碎片包），每4波出现
3. **元素材料3种**：碎片（小怪稀有掉落）、尘埃（小怪稀有掉落+Boss）、结晶（仅Boss掉落）
4. **晋升材料消耗**：基础→进阶 3碎片+1尘埃；进阶→终极 2结晶+5碎片（让两阶段都有材料门槛）
5. **武器经验不归零**：升级时消耗经验但不重置，保留多余经验
6. **元素视觉粒子**：在Enemy.update中按状态生成，控制粒子数量避免性能问题
