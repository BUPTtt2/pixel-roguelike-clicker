**新增executeBossShopPurchase方法**（替代claimBossReward的购买逻辑，保留claimBossReward作为免费奖励处理）：

```javascript
executeBossShopPurchase(id, cost) {
    if (cost > 0) this.runtime.gold -= cost;
    switch (id) {
        case 'promote':
            const wt = this.runtime.weaponType;
            const wlv = this.weaponLevels[wt] || 1;
            const promo = this.runtime.weaponPromotions[wt] || 0;
            if (wlv >= 3 && promo === 0) {
                this.runtime.elements.shard -= 3;
                this.runtime.elements.dust -= 1;
                this.runtime.weaponPromotions[wt] = 1;
                this.showCenterText(`⚔ 晋升成功！${WeaponFactory.info(wt).name} 进阶!`, '#ff4466', 1500);
            } else if (wlv >= 6 && promo === 1) {
                this.runtime.elements.crystal -= 2;
                this.runtime.elements.shard -= 5;
                this.runtime.weaponPromotions[wt] = 2;
                this.showCenterText(`⚔ 晋升成功！${WeaponFactory.info(wt).name} 终极!`, '#ff4466', 1500);
            }
            this.particles.explosion(GW / 2, GH / 2, 40, 0xff4466);
            this.audio.play('levelup');
            break;
        case 'crystal':
            this.runtime.elements.crystal++;
            this.showCenterText('获得元素结晶!', '#ff8866');
            break;
        case 'shard3':
            this.runtime.elements.shard += 3;
            this.showCenterText('获得3个元素碎片!', '#00ffaa');
            break;
        case 'heal':
            this.runtime.heal(Math.floor(this.runtime.maxHp * 0.6));
            this.particles.healEffect(GW / 2, GH / 2 - 20);
            break;
        case 'shield':
            this.runtime.shield += 80;
            break;
        case 'rage':
            this.rageMode = true;
            this.rageTimer = 6000;
            break;
        case 'atk_gem':
            this.runtime.addBonus('atk', 0.05);
            break;
        case 'spec':
            const specs = this.getRandomSpecializations(2);
            specs.forEach(s => this.runtime.specLevels[s.id]++);
            this.runtime.markSpecDirty();
            this.runtime.updateMaxHp();
            break;
        case 'free':
            // 随机三选一
            const r = Math.random();
            if (r < 0.4) {
                this.addWeaponExp(this.runtime.weaponType, 5);
                this.showCenterText('武器经验 +5!', '#ffaa00');
            } else if (r < 0.7) {
                this.runtime.heal(Math.floor(this.runtime.maxHp * 0.4));
                this.particles.healEffect(GW / 2, GH / 2 - 20);
                this.showCenterText('生命恢复!', '#ff4488');
            } else {
                const minionTypes = ['light', 'shadow', 'fire', 'ice', 'lightning'];
                const avail = minionTypes.filter(m => !this.saveData.unlockedMinions.includes(m));
                if (avail.length > 0) {
                    const nm = avail[Math.floor(Math.random() * avail.length)];
                    this.saveData.unlockedMinions.push(nm);
                    const m = new Minion(this, nm);
                    m.spawn(this.playerWorldX, this.playerWorldY);
                    this.minions.push(m);
                    this.showCenterText('解锁新随从!', '#44ff88');
                } else {
                    this.runtime.specLevels.atk = (this.runtime.specLevels.atk || 0) + 2;
                    this.runtime.updateMaxHp();
                    this.showCenterText('攻击专精+2!', '#ffaa00');
                }
            }
            break;
    }
    this.audio.play('buy');
    this.cameras.main.flash(200, 255, 255, 100);
}
```

## 五、假设与决策

1. **假设**：Boss商店可重复购买（除免费奖励外），让玩家用完金币——若用户希望每件商品限购1次，需调整（当前设计为可重复购买）
2. **假设**：武器晋升在Boss商店直接执行（不再需要先在addWeaponExp中材料检查），但addWeaponExp中的材料检查保留作为"经验满但材料不足"的提示
3. **决策**：保留claimBossReward方法（不删除），避免其他地方调用导致报错；新executeBossShopPurchase方法独立处理商店购买
4. **决策**：元素视觉粒子用现有particles.hit方法（简化实现），若需更精美效果可后续优化
5. **决策**：showBossReward购买后递归调用自身刷新商店状态（金币显示、卡片可用性），避免手动刷新复杂逻辑
6. **决策**：免费奖励仍保留三选一随机（武器经验/生命恢复/新随从），让玩家即使金币不足也有保底收益

## 六、风险与注意事项

1. **递归调用showBossReward**：购买后销毁uiElements再调用自身，需确保uiElements数组完整（包含所有卡片和装饰），否则可能残留UI元素。实施时仔细检查uiElements.push覆盖所有创建的元素。
2. **canPromote逻辑**：需同时满足等级条件和材料条件，若材料不足卡片应置灰（enabled=false）。
3. **Merchant.executeAction方法名**：需读取L1500-1550确认实际方法名，可能是buy方法或executeAction方法。
4. **Enemy字段名**：模块E的元素视觉粒子依赖Enemy实例的字段名（bloodLossDps等），需读取Enemy类确认实际字段名。
