# 《像素地牢点击》作品说明文档

## 一、作品名称

**《像素地牢点击》(Pixel Roguelike Clicker)**

## 二、解决的核心业务痛点

在主流游戏中，硬核动作 Roguelike（如《死亡细胞》《以撒》）上手门槛高、碎片化时间难利用；休闲点击类游戏（如《点击英雄》）又过于单调、缺乏挑战。本作品瞄准中间空白：

1. **上手门槛 vs 深度玩法的矛盾**：玩家可纯点击游玩，但 BOSS 阶段、技能预警、宠物走位等机制奖励主动操作
2. **碎片化时间 vs 游戏节奏**：单局 3-8 分钟，自带暂停/继续
3. **AI 协作开发的可交付性**：纯代码生成像素美术，无外部资源依赖，可一键部署到 GitHub Pages

## 三、核心玩法/功能亮点

### 1. 八章节 BOSS 主题剧情
- 8 个独特 BOSS：哥布林王 → 骷髅领主 → 暗影龙 → 深渊魔王 → 时空守护者 → 元素领主 → 虚空先知 → 暗影神
- 每个 BOSS 有独立的视觉、6+ 招技能、阶段切换（破段动画、多血条指示器 ■■◆□□ 2/5）
- 每章节切换主题背景色（绿/灰/紫/红/蓝/橙/深紫/黑红）

### 2. 复杂 BOSS 技能系统
- **统一预警模板**：每个技能都有 0.5-1.5 秒预警（地面红圈/激光线），玩家必须走位躲避
- **阶段化攻击模式**：
  - 简单：单技能，2 个招式
  - 普通：完整技能池
  - 困难：25% 概率组合技
  - 地狱：40% 概率三连击 + 隐藏第四阶段
- **专属技能**：金币雨（哥布林）、骨刺激光（骷髅）、龙息（暗影龙）、湮灭终极（暗影神）等

### 3. 五大宠物系统
- 光明/暗影/火焰/寒冰/闪电 5 种宠物，环绕玩家飞行并自动攻击
- 元素效果：燃烧、减速、连锁闪电
- 每个宠物有独特的视觉和攻击动画

### 4. 武器晋升与永久解锁
- 5 种武器（剑/锤/弓/法杖/匕首）
- 武器晋升系统（★/★★），武器形态 I-IV 变化
- 商店系统：武器晋升、血瓶、护盾、攻击宝石、随机专精

### 5. 符文阵 / 宝箱 / 任务系统
- 固定位置符文阵：需接近（55px）激活，召唤守卫怪物完成挑战
- 固定位置宝箱：需接近（65px）才能拾取，降低刷新频率到 45 秒
- 主线任务 + 支线任务（采集、击杀、探索、护送）

### 6. 存档与难度选择
- 简单 / 普通 / 困难 / 地狱 4 难度，难度越高 BOSS 血量/伤害/速度倍率越大
- 死亡保留/重置存档的双重确认机制
- 商店"应急护盾"急救按钮

## 四、核心创意设计

1. **纯代码生成像素美术**：所有角色/BOSS/武器/宠物/场景元素都用 Phaser Graphics + Canvas 动态绘制，无任何外部图片资源（避免 API 限制，可靠部署）
2. **剧情 1.5 秒极速触发**：游戏开始 1.5 秒内即弹出剧情对话框，NPC 头像 + 大字号 + 角标 + 淡入动画，强化沉浸感
3. **AI 协作生成关卡**：BOSS 数量、强度、技能组合由 AI 动态平衡，避免人工调参
4. **方向性箭头寻路**：地图大（3500×3500），任务目标在屏幕外时显示方向箭头
5. **暂停冻结时间与物理**：避免暂停后怪物继续移动/攻击导致进度跳跃

## 五、AI 协作开发过程

### 1. 整体流程
本作品 100% 通过 AI 协作完成，涵盖架构设计、代码生成、Bug 修复、视觉调优、文档输出全流程。

### 2. 各功能模块的 AI 开发方式

| 功能模块 | AI 协作方式 |
|---|---|
| **架构设计** | AI 设计出 `Boss / Enemy / Pet / Projectile / Particle` 等独立类的分层架构，每个类职责单一 |
| **像素美术** | AI 用 Phaser `Graphics.fillRect()` + `fillCircle()` + `lineStyle()` 逐像素绘制角色/BOSS/武器/宠物（约 200+ 个精灵），所有纹理代码生成 |
| **技能系统** | AI 设计出 `skillWithWarning()` 通用模板（预警 + 效果 + 屏幕震动），复用 8 个 BOSS 的 48+ 个技能 |
| **BOSS 行为树** | AI 为每个 BOSS 设计状态机：WARN → ACTIVE → BREAK → DEATH，控制破段动画、阶段切换 |
| **物理/碰撞** | AI 实现基于距离判断的 AABB 碰撞，替代 Phaser 内置 Arcade Physics 以获得更精细控制 |
| **难度平衡** | AI 调参血量倍率、伤害倍率、技能冷却、技能组合概率，让简单/普通/困难/地狱有明显梯度 |
| **调试/修复** | AI 通过代码静态分析 + 浏览器 Playwright 工具回放，定位 BOSS 残留、UI 叠字、宠物不动等 20+ 个 Bug |
| **Git 部署** | AI 自动 git commit + git push 到 GitHub Pages，添加 `.nojekyll` 绕过 Jekyll 模板 |
| **文档输出** | AI 生成 README、SPEC 文档（本作品说明），自动 commit 到 docs/ 目录 |

### 3. AI 在保证游戏品质方面的核心价值

1. **极快迭代速度**：每个功能调整 30 秒内完成，从需求到上线路径短
2. **跨模块一致性**：AI 维护全局设计语言（美术风格、UI 字号、配色、命名），避免碎片化
3. **Bug 模式识别**：AI 通过错误现象反推根因（如 BOSS 残留 = 无限 tween 阻止 container 销毁）
4. **代码可维护性**：AI 主动识别并修复技术债（如 setTimeout 改 Phaser delayedCall 防内存泄漏）
5. **数据驱动平衡**：AI 调参基于游戏内事件（连击数、击杀数、死亡时间），而非凭感觉
6. **文档与代码同步**：AI 在改代码的同时自动更新设计文档（specs/），保持文档不过期

### 4. 关键 AI 决策亮点

- **解耦的 BOSS UI**：将血条/名字从 container 抽出独立管理，根除"换 BOSS 后旧 UI 残留"
- **BossManager 单例**：所有 BOSS 通过唯一 manager 管理，强制 spawn 前清理，防止多个 BOSS 共存
- **宠物轨道运动**：从"平滑插值"改为"直接 setPosition + 旋转角速度"，彻底解决"宠物像贴图不动"
- **forceDestroy 统一销毁**：先 `killTweensOf` 再 destroy 子元素，根除 tween 持有引用导致清理失败
